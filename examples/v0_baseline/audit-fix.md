对照 plan.md 执行穷尽式审计-修复一体化循环。不要只做计划，立即执行修复。

## 审计维度

对每个维度逐一检查：

1. **完整性**: plan.md 中要求的每个分析步骤是否都有对应结果文件
2. **数据准确性**: 结果文件中的数据是否与原始数据/中间文件一致（抽样验证关键数值）
3. **分析准确性**: 统计方法、参数设置、阈值选择是否正确
4. **方法合理性**: 使用的工具/流程/参考基因组版本是否合适
5. **图表-数据一致性**: 图表中展示的数据是否与结果文件中的数据吻合
6. **基因组版本**: 使用的参考基因组/注释版本是否与 plan.md 要求一致

## 问题分级

- **P0 — 数据完整性**: 缺失结果、数据丢失、关键文件损坏
- **P1 — 方法正确性**: 参数错误、工具版本不对、基因组版本不匹配
- **P2 — 完整性缺失**: 部分分析未完成但不影响核心结论
- **P3 — 润色优化**: 图表美化、报告格式、命名规范

## 执行流程

### 每一轮循环：

1. **审计**: 使用 bio-result-auditor agent 全面审计，输出问题清单
2. **分级**: 按 P0→P3 排序所有发现的问题
3. **修复**: 立即修复所有问题，P0 优先
   - 缺失的分析：重新运行脚本
   - 数据不一致：追溯源头修正
   - 图表问题：重新生成
   - 代码问题：直接修改脚本
4. **提交**: `git add -A && git commit -m "audit-fix round N: 修复内容摘要"`
5. **重新审计**: 验证修复是否生效，检查是否引入新问题

### 循环控制

- 最多执行 **5 轮**循环
- 当所有 P0/P1 问题清零且无新问题时提前终止
- 每轮开始前输出当前轮次和剩余问题数
